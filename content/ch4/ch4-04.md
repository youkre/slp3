## 4.4 多项逻辑回归

有时我们需要处理超过两个类别的分类问题。
例如，可能需要进行三类情感分类（正面、负面或中性）。
或者，可能会分配第 17 章将要介绍的一些标签，比如一个词的词性（从 10个、30 个甚至 50 个不同的词性中选择），或一个短语的命名实体类型（从“人名”、“地点”、“组织”等标签中选择）。
或者在大语言模型中，词汇表中 |V| 个词，预测下一个词其中的哪一个，这就是 |V| 分类。

在这种情况下，使用**多项逻辑回归**（multinomial logistic regression），也被称为**Softmax回归**（在早期的NLP文献中，有时会看到它被称为**最大熵分类器**，maxent classifier）。
在多项逻辑回归中，我们希望将每个观测样本标记为 <span class="math">\(K\)</span> 个类别中的某一个 <span class="math">\(k\)</span>，并规定只有一个类别是正确的（有时称为**硬分类**；一个观测样本不能同时属于多个类别）。
我们采用以下表示方式：每个输入 <span class="math">\(\mathbf{x}\)</span> 的输出 <span class="math">\(\mathbf{y}\)</span> 将是一个长度为 <span class="math">\(K\)</span> 的向量。
如果类别 <span class="math">\(c\)</span> 是正确的类别，我们将 <span class="math">\(y_c\)</span> 设为1，并将 <span class="math">\(\mathbf{y}\)</span> 的所有其他元素设为0，即 <span class="math">\(y_c = 1\)</span> 且 <span class="math">\(y_j = 0 \quad \forall j \neq c\)</span>。
像这样的向量 <span class="math">\(\mathbf{y}\)</span>（只有一个值为1，其余为0）被称为**独热向量**（one-hot vector）。
分类器的任务是生成一个估计向量 <span class="math">\(\hat{\mathbf{y}}\)</span>。
对于每个类别 <span class="math">\(k\)</span>，值 <span class="math">\(\hat{y}_k\)</span> 将是分类器对概率 <span class="math">\(p(y_k = 1|\mathbf{x})\)</span> 的估计。



多项逻辑分类器使用 Sigmoid 函数的一个推广版本，称为**Softmax函数**，来计算 <span class="math">\(p(y_k = 1|\mathbf{x})\)</span>。
Softmax函数接收一个包含 <span class="math">\(K\)</span> 个任意值的向量 <span class="math">\(\mathbf{z} = [z_1, z_2, ..., z_K]\)</span>，并将其映射为一个概率分布，其中每个值都在 <span class="math">\([0,1]\)</span> 范围内，且所有值的总和为1。
与Sigmoid函数一样，它也是一种指数函数。

对于一个维度为 <span class="math">\(K\)</span> 的向量 <span class="math">\(\mathbf{z}\)</span>，Softmax函数定义如下：

<div class="math">
$$
\mathrm{softmax}(z_i) = \frac{\exp(z_i)}{\sum_{j=1}^K \exp(z_j)} \quad 1 \leq i \leq K
\tag{4.16}
$$
</div>

因此，输入向量 <span class="math">\(\mathbf{z} = [z_1, z_2, \cdots, z_K]\)</span> 的Softmax结果本身也是一个向量：

<div class="math">
$$
\mathrm{softmax}(\mathbf{z}) = \left[\frac{\exp(z_1)}{\sum_{i=1}^K \exp(z_i)}, \frac{\exp(z_2)}{\sum_{i=1}^K \exp(z_i)}, \cdots, \frac{\exp(z_K)}{\sum_{i=1}^K \exp(z_i)}\right]
\tag{4.17}
$$
</div>

分母 <span class="math">\(\sum_{i=1}^K \exp(z_i)\)</span> 用于将所有值归一化为概率。
例如，给定向量：

<div class="math">
$$
\mathbf{z} = [0.6, 1.1, -1.5, 1.2, 3.2, -1.1]
$$
</div>

其对应的（四舍五入后的）Softmax(<span class="math">\(\mathbf{z}\)</span>) 结果为：

<div class="math">
$$
[0.05, 0.09, 0.01, 0.10, 0.74, 0.01]
$$
</div>

与 Sigmoid 函数类似，Softmax 函数也具有将极端值压缩至0或1的特性。
因此，如果其中一个输入值远大于其他输入值，Softmax 会倾向于将其概率推高至接近1，同时抑制较小输入值的概率。

最后，请注意，得分向量 <span class="math">\(\mathbf{z}\)</span> 是 softmax 函数的输入。与 Sigmoid 函数的情况类似，我们将 <span class="math">\(\mathbf{z}\)</span> 称为 **logits**（参见公式 4.7）。



当我们将 Softmax 应用于逻辑回归时，其输入（与 Sigmoid 函数类似）将是权重向量 <span class="math">\(\mathbf{w}\)</span> 与输入向量 <span class="math">\(\mathbf{x}\)</span> 的点积（再加上一个偏置项）。
但现在，我们需要为 <span class="math">\(K\)</span> 个类别中的每一个都设置独立的权重向量 <span class="math">\(\mathbf{w}_k\)</span> 和偏置项 <span class="math">\(b_k\)</span>。
因此，每个输出类别 <span class="math">\(\hat{y}_k\)</span> 的概率可以计算如下：

<div class="math">
$$
p(y_k = 1|\mathbf{x}) = \frac{\exp(\mathbf{w}_k \cdot \mathbf{x} + b_k)}{\sum_{j=1}^K \exp(\mathbf{w}_j \cdot \mathbf{x} + b_j)}
\tag{4.18}
$$
</div>

公式 4.18 的形式似乎暗示我们需要分别计算每个输出。
然而，为了更高效地利用现代向量化硬件进行计算，我们通常会以另一种方式构建方程。
为此，我们将 <span class="math">\(K\)</span> 个权重向量表示为一个权重矩阵 <span class="math">\(\mathbf{W}\)</span>，并将偏置项表示为一个偏置向量 <span class="math">\(\mathbf{b}\)</span>。
<span class="math">\(\mathbf{W}\)</span> 的每一行 <span class="math">\(k\)</span> 对应于权重向量 <span class="math">\(\mathbf{w}_k\)</span>。
因此，<span class="math">\(\mathbf{W}\)</span> 的形状为 <span class="math">\([K \times f]\)</span>，其中 <span class="math">\(K\)</span> 是输出类别的数量，<span class="math">\(f\)</span> 是输入特征的数量。
偏置向量 <span class="math">\(\mathbf{b}\)</span> 则包含 <span class="math">\(K\)</span> 个输出类别各自的偏置值。
以这种方式表示权重后，我们可以用一个简洁的公式一次性计算出 <span class="math">\(\hat{\mathbf{y}}\)</span>，即所有 <span class="math">\(K\)</span> 个类别的输出概率向量：

<div class="math">
$$
\hat{\mathbf{y}} = \mathrm{softmax}(\mathbf{Wx} + \mathbf{b})
\tag{4.19}
$$
</div>

如果你仔细推导一下矩阵运算过程，就会发现第一个输出类别 <span class="math">\(\hat{y}_1\)</span> 的估计得分（在应用 Softmax 之前）确实等于 <span class="math">\(\mathbf{w}_1 \cdot \mathbf{x} + b_1\)</span>。

一种有助于解读权重矩阵 <span class="math">\(W\)</span> 的方式是，当每一行 <span class="math">\(w_k\)</span> 当作类别 <span class="math">\(k\)</span> 的**原型**（prototype）。
学习到权重向量 <span class="math">\(w_k\)</span> 把类别表示为一种模版。
由于两个相似度高的向量，它们的点积结果更高，点积相当于相似度函数。
因此，逻辑斯谛回归学习了每一类别的典型表示，结果在 <span class="math">\(K\)</span> 个类别中，输入的向量被赋予与其最相似的类别 <span class="math">\(k\)</span>。

对权重矩阵 <span class="math">\(\mathbf{W}\)</span> 的一种有益理解是：将每一行 <span class="math">\(\mathbf{w}_k\)</span> 视为类别 <span class="math">\(k\)</span> 的一个**原型**（prototype）。
所学习到的权重向量 <span class="math">\(\mathbf{w}_k\)</span> 以某种模板的形式表征该类别。
由于两个向量越相似，它们的点积就越大，因此点积可充当一种相似度函数。
由此，逻辑回归实际上是在为每个类别学习一个**范例式**（exemplar）表示，使得输入向量被分配给 <span class="math">\(K\)</span> 个类别中与其最相似的那个类别 <span class="math">\(k\)</span>（Doumbouya et al., 2025）。

多项逻辑回归中的特征与二元逻辑回归中的特征作用类似，唯一的区别是如上所述，我们需要为 <span class="math">\(K\)</span> 个类别中的每一个都设置独立的权重向量和偏置项。回顾第 66 页提到的二元分类中的感叹号特征 <span class="math">\(x_5\)</span>：

<div class="math">
$$
x_5 =
\begin{cases}
1 & \text{if “!” } \in \text{doc} \\
0 & \text{otherwise}
\end{cases}
$$
</div>

在二元分类中，某个特征上的正权重 <span class="math">\(w_5\)</span> 会促使分类器倾向于将样本判为 <span class="math">\(y = 1\)</span>（正面情感），而负权重则会使其倾向于 <span class="math">\(y = 0\)</span>（负面情感），其绝对值大小表示该特征的重要性。
而在多项逻辑回归中，由于每个类别都有独立的权重，因此一个特征可能对某个类别是支持证据，而对另一个类别则是反对证据。

例如，在三类情感分类任务中，我们需要将每篇文档分配到三个类别之一：+（正面）、−（负面）或 0（中性）。
此时，与感叹号相关的特征可能对“中性”类别具有负权重，而对“正面”或“负面”类别具有正权重：

| 特征 | 定义 | <span class="math">\(w_{5,+}\)</span> | <span class="math">\(w_{5,-}\)</span> | <span class="math">\(w_{5,0}\)</span> |
| :--- | :---: | :---: | :---: | :---: |
| <span class="math">\(f_5(x)\)</span> | <span class="math">\(\begin{cases} 1 & \text{if “!” } \in \text{doc} \\ 0 & \text{otherwise} \end{cases}\)</span> | 3.5 | 3.1 | -5.3 |

由于这些特征权重同时依赖于输入文本和输出类别，我们有时会明确表示这种依赖关系，将特征本身写成 <span class="math">\(f(x, y)\)</span>：即输入和类别的函数。
使用这种记法，上面的 <span class="math">\(f_5(x)\)</span> 可以表示为三个特征：<span class="math">\(f_5(x,+)\)</span>、<span class="math">\(f_5(x,−)\)</span> 和 <span class="math">\(f_5(x,0)\)</span>，每个特征对应一个单一的权重。
我们将在第 17 章描述条件随机场（CRF）时使用这种记法。
