# 6.4 前馈网络在自然语言处理中的应用：分类任务

现在我们来看如何将前馈网络应用于 NLP 分类任务。
实际上，简单的前馈网络并不是当前文本分类的主流方法；在真实应用中，我们会使用更先进的架构，例如第 10 章介绍的 BERT 等 Transformer 模型。<br>
尽管如此，通过构建一个基于前馈网络的文本分类器，我们可以引入若干核心概念，这些概念贯穿全书，包括：嵌入矩阵（embedding matrix）、表示池化（representation pooling）和表示学习（representation learning）。

但在介绍这些概念之前，我们先从一个最简单的分类器开始——仅对第 4 章的情感分类器做最小改动。
与第 4 章一样，我们仍使用人工设计的特征，将其送入分类器，并输出类别概率。
唯一的区别是，我们将分类器从逻辑回归替换为神经网络。



我们从一个简单的两层情感分类器入手：以第 4 章的逻辑回归分类器（对应单层网络）为基础，仅增加一个隐藏层。

输入元素 <span class="math">\(\mathbf{x}_i\)</span> 可以是图 4.2 中那样的标量特征，例如<span class="math">\(\mathbf{x}_i\)</span> = 文档中的总词数，<span class="math">\(\mathbf{x}_2\)</span> = 文档中积极情感词典词的出现次数，如果文档包含“no” 则 <span class="math">\(\mathbf{x}_3\)</span> = 1，依此类推，共 <span class="math">\(d\)</span> 个特征。
输出层 <span class="math">\(\hat{y}\)</span> 可以有两个节点（分别对应正面、负面情感），或三个节点（正面、负面、中性）。此时 <span class="math">\(\hat{y}_1\)</span> 表示正面情感的概率，
<span class="math">\(\hat{y}_2\)</span> 表示负面情感的概率，<span class="math">\(\hat{y}_3\)</span> 表示中性情感的概率。
整个模型的计算公式与前述两层网络完全一致（如前所述，我们继续用 <span class="math">\(\sigma\)</span> 泛指任意非线性激活函数，无论是 Sigmoid、ReLU 还是其他）：

<div class="math">
$$
\begin{align*}
\mathbf{x} &= [\mathbf{x}_i,\mathbf{x}_2,...\mathbf{x}_d ] (\text{each} \mathbf{x}_i \text{is a hand-designed feature}) \\
\mathbf{h} &= \sigma(\mathbf{Wx} + \mathbf{b}) \\
\mathbf{z} &= \mathbf{Uh} \\
\hat{\mathbf{y}} = \mathrm{softmax}(\mathbf{z})
\tag{6.19}
\end{align*}
$$
</div>

图 6.10 展示了该架构的示意图。
如前所述，在逻辑回归分类器中加入这个隐藏层，使网络能够捕捉特征之间的非线性交互关系。
仅此一点就可能带来性能更好的情感分类器。

公式 6.19 描述的是对单个样本 <span class="math">\(\mathbf{x}\)</span> 的分类过程，在实际应用中，我们希望高效地对包含 <span class="math">\(m\)</span> 个样本的整个测试集进行分类。
为此，我们采用向量化（vectorizing）方法，这与逻辑回归中的做法完全一致：避免使用 for 循环逐个处理样本，而是通过矩阵运算一次性完成整个测试集的计算。
具体而言，我们将每一个输入 <span class="math">\(x\)</span> 的所有输入特征向量打包成一个输入矩阵 <span class="math">\(\mathbf{X}\)</span>，其中第 <span class="math">\(i\)</span> 行是一个行向量，表示第 <span class="math">\(i\)</span> 个输入样本 <span class="math">\(x^{(i)}\)</span> 的特征（即 <span class="math">\(\mathbf{x}^{(i)}\)</span> 本身）。
若每个输入的特征维度为 <span class="math">\(d\)</span>，则 <span class="math">\(\mathbf{X}\)</span> 的形状为 <span class="math">\([m \times d]\)</span>。

由于我们现在将每个输入表示为行向量（而非列向量），需对公式 (6.19) 稍作调整。
<span class="math">\(\mathbf{X}\)</span> 的形状为 <span class="math">\([m \times d]\)</span>，权重矩阵 <span class="math">\(\mathbf{W}\)</span> 的形状为 <span class="math">\([d_h \times d]\)</span>（<span class="math">\(d_h\)</span> 为隐藏层维度），我们需要调整 <span class="math">\(\mathbf{X}\)</span> 和 <span class="math">\(\mathbf{W}\)</span> 乘法顺序，转置 <span class="math">\(\mathbf{W}\)</span>，才能使矩阵乘法合法并得到形状为 <span class="math">\([m \times d_h]\)</span> 的隐藏层输出矩阵 <span class="math">\(\mathbf{H}\)</span>。[^1]

[^1]: 如果我们坚持将每个样本作为列向量存储（即 <span class="math">\(\mathbf{X}\)</span> 形状为 <span class="math">\([d \times m]\)</span>），则可保留原始乘法顺序 <span class="math">\(\mathbf{W} \mathbf{X}\)</span>。但将样本作为行向量处理在神经网络中更为常见且方便。

此外，原偏置向量 <span class="math">\(\mathbf{b}\)</span> 的形状是 <span class="math">\([1 \times d_h]\)</span>），现在需被广播为形状 <span class="math">\([m \times d_h]\)</span> 的矩阵。
同理，下一步计算中也需要对 <span class="math">\(\mathbf{U}\)</span> 转置。
最终，输出矩阵 <span class="math">\(\hat{\mathbf{Y}}\)</span> 的形状为 <span class="math">\([m \times 3]\)</span>（或更一般地 <span class="math">\([m \times d_o]\)</span>，其中 <span class="math">\(d_o\)</span> 为输出类别数），其第 <span class="math">\(i\)</span> 行即为第 <span class="math">\(i\)</span> 个样本的输出概率向量 <span class="math">\(\hat{\mathbf{y}}^{(i)}\)</span>。
计算整个测试集输出类别分布的最终计算公式如下：

<div class="math">
$$
\begin{align*}
\mathbf{H} &= \sigma(\mathbf{XW}^T + \mathbf{b}) \\
\mathbf{Z} &= \mathbf{HU}^T \\
\hat{\mathbf{Y}} &= \mathrm{softmax}(\mathbf{Z})
\tag{6.20}
\end{align*}
$$
</div>

在本书中，有时我们会看到矩阵乘法写成 <span class="math">\(\mathbf{WX} + \mathbf{b}\)</span>，有时写成 <span class="math">\(\mathbf{XW} + \mathbf{b}\)</span>。因此，在任何公式中，都要清楚权重矩阵的形状，这一点始终至关重要。
